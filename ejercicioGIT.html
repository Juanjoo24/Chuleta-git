Examen Git y Github

Alumn@: Juan José Hoyos Castrillón

  1. ¿Que es un sistema de control de versiones?

    [El control de versiones sincroniza las versiones y garantiza que los cambios no entren en conflicto con los cambios de otros usuarios.]

  2. ¿Cual es la diferencia entre git y github?

    [Git es la herramienta para realizar el seguimiento de los cambios, mientras que GitHub es el lugar para almacenar los proyectos y trabajar en equipo]

  3. ¿Cómo creo una rama en git?

    [Para crear una rama, use el comando git branch]

  4. ¿Cómo clono un repositorio?

    [Para crear un repositorio por medio de comandos mkdir]

  5. ¿Cómo puedo bajar los ultimos cambios de un repositorio?

    [Mediante el comando git pull]

  6. ¿Cómo puedo gener un tag?

    [ Para crear un tag por medio de comandos git tag "nombre del tag"]

  7. Cómo hago una mezcla de ramas?

    [con git checkout para cambiar a la rama en la que desea fusionar. depues usamos git merge y especificamos el nombre de la otra rama que desea incorporar a esta rama ]

  8. ¿Cómo puedo moverme a un commit especifico?

    [Para movernos a un commit específico por medio de comandos git checkout "hash del commit"]

  9. ¿Cómo puedo borrar una rama?

    [Para borrar una rama por medio de comandos git branch -d "nombre de la rama"]

  10. ¿Cómo puedo borrar un tag?

    [Para borrar el tag por medio de comandos git tag -d "nombre del tag"]

  11. ¿Cómo puedo ver los cambios que estan por enviarse al repositorio?

    [con el git status]

  12. ¿Para que sirve el comando git remote -v?

    [Para ver repositorios remotos asociados a mi repositorio local]

  13. ¿Cómo le digo a git cual es mi email y correo?

    [Para tu nomnbre : git config user.name "nombre" Para correo : git config user.email "tu correo"]

  14. ¿Cuales son los pasos para conectar git con github?

    [debemos de poner nuestro nombre y nuestro correo, tras esto iniciaremos sesión en git y crearemos nuestro repositorio, el cual será vinculado y debemos vicular nuestra clave priavada el github y tras esto ya podemos trabajar con él]

  15. ¿Cómo puedo crear un alias en git?

    [git config -- global alias.st "status" con este ejemplo en vez de utilizar git status podemos poner de forma más rápida git st]
